\doxysubsection{Detailed Description}
\subsubsection*{template$<$std\+::size\+\_\+t dim$>$\newline
class tracktable\+::domain\+::feature\+\_\+vectors\+::\+Feature\+Vector$<$ dim $>$}
Point type for feature vectors. 

A feature vector is a list of numbers that collectively describe properties of some entity, generally a trajectory. We typically treat feature vectors as a kind of fingerprint\+: we don\textquotesingle{}t want to modify them or do arithmetic with them, but instead look at the structure of a larger collection of feature vectors.

In Tracktable we use the {\ttfamily \doxylink{classtracktable_1_1domain_1_1feature__vectors_1_1_feature_vector}{Feature\+Vector}} template for such things. You can specify any dimension you want from 1 on up. Algorithms such as {\ttfamily DBSCAN} and the {\ttfamily R-\/tree} are templated on point type so that you can use them with any kind of feature vector you want.
