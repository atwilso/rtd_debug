\doxysubsection{Detailed Description}
\subsubsection*{template$<$class Base\+PointT$>$\newline
class tracktable\+::\+Trajectory\+Point$<$ Base\+Point\+T $>$}
Add object ID, timestamp, property map. 

This class will add trajectory properties (a timestamp, an object ID and storage for named properties) to any point class.

\doxylink{namespacetracktable_a682e5fb98918f65770834e07ff51d21c}{Timestamp} is a {\ttfamily \doxylink{namespacetracktable_a682e5fb98918f65770834e07ff51d21c}{tracktable\+::\+Timestamp}} which (under the hood) is a {\ttfamily boost\+::posix\+\_\+time\+::ptime}. Object ID is stored as a string.

We also include an interface to set, get and enumerate arbitrary named properties. The only restriction is that the types of these properties are limited to timestamps, floating-\/point numbers and strings. If you need something more flexible than that please consider creating your own alternative point class either by subclassing \doxylink{classtracktable_1_1_trajectory_point}{Trajectory\+Point} or by composition.

\begin{DoxyNote}{Note}
Named property support is implemented using {\ttfamily boost\+::variant}. You can either use {\ttfamily boost\+::get\texorpdfstring{$<$}{<}\texorpdfstring{$>$}{>}} to cast it to your desired data type or call one of the {\ttfamily (type)\+\_\+property\+\_\+value} functions to retrieve it with no casting necessary. Take a look at tracktable/\+Core/\+Tests/test\+\_\+trajectory\+\_\+point\+\_\+lonlat.\+cpp (XXX CHECK THIS) for a demonstration. 
\end{DoxyNote}
