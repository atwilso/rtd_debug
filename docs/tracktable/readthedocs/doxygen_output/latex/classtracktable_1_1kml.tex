\doxysubsection{Detailed Description}
Handles writing tracks as kml Has lots of internal structure to make the following use cases work\+: 


\begin{DoxyCode}{0}
\DoxyCodeLine{std::cout\ <<\ kml::header;}
\DoxyCodeLine{std::cout\ <<\ kml(t);}
\DoxyCodeLine{std::cout\ <<\ kml::footer;}
\DoxyCodeLine{}
\DoxyCodeLine{out\ =\ ofstream;}
\DoxyCodeLine{out\ <<\ kml(t);}

\end{DoxyCode}


For both of the above a default style will be used (green, width = 3) in the case of a single trajectory and a random color will be used with the same default \hypertarget{classtracktable_1_1kml_autotoc_md0}{}\doxysubsubsection{\texorpdfstring{width = 3 for a vector of trajectories.}{width = 3 for a vector of trajectories.}}\label{classtracktable_1_1kml_autotoc_md0}
To control color and width\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{std::cout\ <<\ kml::header;}
\DoxyCodeLine{kml::write(std::cout,\ t,\ \textcolor{stringliteral}{"{}AABBGGRR"{}},\ 4)}
\DoxyCodeLine{std::cout\ <<\ kml::footer;}
\DoxyCodeLine{}
\DoxyCodeLine{kml::write(\textcolor{stringliteral}{"{}filname"{}},\ t,\ \textcolor{stringliteral}{"{}AABBGGRR"{}},\ 5)}

\end{DoxyCode}


These are the limits of style control at this time.
