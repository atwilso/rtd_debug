\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename PointT$>$\newline
class tracktable\+::\+Point\+Reader$<$ Point\+T $>$}
Read points from files. 

This reader wraps the following pipeline\+:


\begin{DoxyItemize}
\item Read lines from a text file
\item Skip any lines that begin with a designated comment character (\textquotesingle{}\#\textquotesingle{} by default)
\item Tokenize each line using specified delimiters (whitespace by default)
\item Create a point (user-\/specified type) from each tokenized line
\item Return the resulting points via a C++ iterator
\end{DoxyItemize}

You will use {\ttfamily \doxylink{classtracktable_1_1_point_reader_aaf26666a2c3ee27589cbf0f29e35ac1d}{set\+\_\+input()}} to supply an input stream, {\ttfamily \doxylink{classtracktable_1_1_point_reader_a28e567e4badcabd3d25c29d84258bef9}{set\+\_\+comment\+\_\+character()}} to configure which lines to skip, {\ttfamily set\+\_\+delimiter()} to specify how to turn lines into tokens, and {\ttfamily set\+\_\+column\+\_\+for\+\_\+field()} to assign columns in the data file to fields (object ID, longitude, latitude, etc) on the point.

Take a look at the test case for this class (in C++/\+RW/\+Tests/test\+\_\+integrated\+\_\+trajectory\+\_\+point\+\_\+reader.cpp) for an example of how to use it.
